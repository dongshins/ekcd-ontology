## Overview

**EKCD v1.0.4** is the first public release of the **EKCD (EKC-derived) ontology** line.
It establishes a citable and versioned baseline for a personal derivative ontology that extends the EKC line for ongoing Korean cultural heritage and digital humanities knowledge modeling.

EKCD is not intended to replace EKC.
Rather, it is maintained as a derivative and extensible companion ontology that preserves conceptual continuity with the EKC baseline while providing an additional layer for selected extensions, local usage guidance, and practical dataset-building support.

## Main points of this release

### 1. First public archival baseline of EKCD
This release marks the first public GitHub release of the `ekcd:` ontology line and is intended to serve as the initial reproducible reference point for subsequent EKCD development.

### 2. Derivative relationship to EKC
The ontology imports **EKC 2025** as its baseline and should be understood as an EKC-derived extension line rather than an independent replacement ontology.

### 3. Added date-oriented datatype properties
This release includes date-oriented properties introduced for practical dataset construction and normalization:

- `ekcd:occurrenceDate`
- `ekcd:startDate`
- `ekcd:endDate`
- `ekcd:originalDate`
- `ekcd:accessURL`

### 4. Local guidance for semantic mapping practice
This release includes local operational guidance for external mapping and identity-related properties, especially:

- `owl:sameAs`
- `skos:mappingRelation`
- `skos:exactMatch`
- `skos:closeMatch`

These guidance notes are intended for EKCD-side operational use and should be read as derivative/local modeling policy rather than as institutional EKC core policy.

### 5. Operational handling of web resources
This release also provides local guidance for using `edm:isShownAt` and `edm:isShownBy` together with `ekcd:accessURL`, so that dereferenceable URLs can be recorded on linked WebResource / Multimedia nodes in a more explicit way.

## Compatibility and positioning

- **EKC** remains the institutional baseline ontology.
- **EKCD** is the derivative extension line.
- Datasets may use EKCD as an active TBox while remaining interoperable with the EKC conceptual backbone.
