# EKCD v0.9.7

First public release of **EKCD: EKC Derived Ontology**.

## Highlights
- Establishes **EKCD** as a separate derivative ontology line
- Introduces namespace prefix `ekcd:`
- Adds datatype properties for more explicit temporal modeling and web access information:
  - `ekcd:occurrenceDate`
  - `ekcd:startDate`
  - `ekcd:endDate`
  - `ekcd:originalDate`
  - `ekcd:accessURL`
- Restores **EKC** to its original simplified practice by using `owl:sameAs` as its sole identity/equivalence-oriented linking property
- Extends **EKCD** with additional standard mapping properties for more differentiated link curation:
  - `owl:sameAs`
  - `skos:exactMatch`
  - `skos:closeMatch`
  - `skos:mappingRelation`
- Clarifies the stricter operational distinction among `owl:sameAs`, `skos:exactMatch`, and `skos:closeMatch`
- Supports ongoing Protégé + Cellfie-based dataset construction
- Intended for active use in projects including **국조속오례의**

## Notes
EKCD is a personal derivative ontology line based on EKC.  
It is intended to preserve conceptual continuity with EKC while providing a distinct namespace and release framework for ongoing personal ontology extension work.

In this release, EKC is understood as retaining `owl:sameAs` as its sole property for identity or equivalence-oriented linking, in keeping with its original design. By contrast, EKCD adopts additional standard SKOS mapping properties—`skos:exactMatch`, `skos:closeMatch`, and `skos:mappingRelation`—in order to support a stricter and more explicit distinction between strict identity, near-equivalent correspondence, and looser conceptual similarity.

The newly added datatype properties are intended to support spreadsheet-driven and Cellfie-based RDF production workflows by enabling more explicit recording of single-point dates, temporal ranges, source-form date expressions, and access-related web information for digital or online resources.
