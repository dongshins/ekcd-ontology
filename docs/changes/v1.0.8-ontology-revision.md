# EKCD Ontology v1.0.8 Revision Note

## 1. Purpose

This note documents the ontology-level revision from **EKCD v1.0.7** to **EKCD v1.0.8**.

The revision is a non-breaking maintenance update focused on:

1. clarifying EKCD ontology metadata;
2. recording an explicit Turtle authoring and serialization policy;
3. aligning source/reference metadata with the canonical EKC ontology IRI;
4. preserving compatibility for existing data that uses v1.0.7 classes and properties.

The following RDF-level removals were identified relative to v1.0.7:

```turtle
<http://dh.aks.ac.kr/ontologies/ekcd>
    owl:versionInfo "EKCD v1.0.7"@en ;
    dcterms:modified "2026-05-07"^^xsd:date ;
    dcterms:source <https://github.com/dongshins/EKC_ontology> ;
    rdfs:comment "The EKC Ontology is a domain-specific vocabulary for modeling relationships among historical facts, source documents, persons, and locations in Korean cultural heritage. Initially developed during the Encyclopedic Archives of Korean Culture digitization project at the Academy of Korean Studies and refined through various digital humanities projects."@en .
```

The Korean ontology description literal and the English title literal also changed, so the previous literal values are no longer present.

## 6. Serialization-level change

v1.0.7 included:

```turtle
@base <http://dh.aks.ac.kr/ontologies/ekc#> .
```

v1.0.8 removes this `@base` declaration and records a formal EKCD authoring policy that discourages:

- `@base` declarations;
- the empty prefix `:`;
- relative IRIs such as `<#...>`, `<...>`, and `<>`.

This is a serialization-policy change rather than a vocabulary-breaking change.

## 8. Vocabulary compatibility

No changes were detected in the following vocabulary surfaces:

- OWL class declarations;
- OWL object property declarations;
- EKCD-local datatype property declarations.

Therefore, v1.0.8 should be treated as a **non-breaking patch release** for data that already uses the v1.0.7 vocabulary terms.

## 9. Impact on ABox data

Existing ABox data should not require term replacement. However, maintainers should review Turtle authoring conventions:

| Area | v1.0.8 recommendation |
|---|---|
| Resource IRIs | Use explicit `ekc:` or `ekcd:` prefixed names for EKC/EKCD resources. |
| Empty prefix | Do not use the empty prefix `:`. |
| Relative IRIs | Do not use relative forms such as `<#...>`, `<...>`, or `<>`. |
| URL literals | Keep concrete URLs on target web-resource nodes using `ekcd:accessURL`. |
| Display links | Keep `edm:isShownAt` and `edm:isShownBy` as object properties. |
| Date expressions | Use `ekcd:occurrenceDate`, `ekcd:startDate`, `ekcd:endDate`, and `ekcd:originalDate` according to single-date/period/original-source needs. |

## 10. Recommended release checklist

Before publishing the GitHub release:

- [ ] Confirm that the English `dc:title` literal is intentionally written as released.
- [ ] Confirm that `CITATION.cff` and `.zenodo.json` both use version `1.0.8`.
- [ ] Upload or archive the v1.0.8 ontology file with the release.
- [ ] If Zenodo is used, update the GitHub Release and README with the assigned DOI after publication.
