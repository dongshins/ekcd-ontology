- Version: v1.1.2

- Zenodo concept DOI: https://doi.org/10.5281/zenodo.19753664

## 1. Purpose

This note documents the ontology-level revision from **EKCD v1.0.8** to **EKCD v1.1.2**.

The revision is a vocabulary-expanding, non-breaking minor update focused on:

1. adding classes and properties for interpretive assertions;
2. adding controlled confidence-level values for correspondence and contextual association;
3. adding evidence-tracking properties for interpretive modeling;
4. adding utterance modeling classes and properties as event-level extensions of the EKC event model;
5. recording Su-jeong PARK as an EKCD contributor;
6. preserving compatibility for existing data that uses v1.0.8 classes and properties.

### 4.5 New controlled confidence-level individuals

v1.1.2 adds three controlled values for interpretive confidence. Each is typed as an `owl:NamedIndividual`, an `ekcd:InterpretiveConfidenceLevel`, and a `skos:Concept`.

| Term | English label | Korean label | Intended use |
|---|---|---|---|
| `ekcd:Direct` | Direct correspondence | 직접 대응 | Directly supported correspondence. |
| `ekcd:Probable` | Probable correspondence | 유력한 대응 | Strongly inferred correspondence supported by multiple pieces of evidence. |
| `ekcd:Contextual` | Contextual association | 맥락적 연관 | Broader contextual association weaker than direct or probable correspondence. |

## 6. Serialization-level change

v1.1.2 preserves the serialization policy introduced in v1.0.8.

Both compared files avoid:

- `@base` declarations as Turtle directives;
- the empty prefix `:`;
- relative IRIs such as `<#...>`, `<...>`, and `<>`.

The existing `ekcd:authoringPolicy` annotation is retained:

```turtle
ekcd:authoringPolicy
    "EKCD TBox files and related ABox data do not use @base declarations, the empty prefix ':', or relative IRIs such as <#...>, <...>, and <>. EKC and EKCD resources must be explicitly written with the ekc: or ekcd: prefixed names."@en ,
    "EKCD TBox 및 관련 ABox 데이터에서는 @base 선언, empty prefix ':', 상대 IRI(<#...>, <...>, <>)를 사용하지 않도록 합니다. EKC 및 EKCD 자원은 반드시 ekc: 또는 ekcd: prefixed name으로 명시하도록 합니다."@ko .
```

This is a preserved serialization-policy convention rather than a new serialization-level change.

## 8. Vocabulary compatibility

v1.1.2 expands the EKCD vocabulary surface. Unlike v1.0.8, which was primarily a metadata and serialization-policy patch, v1.1.2 adds new modeling terms.

No removals were detected in the following v1.0.8 vocabulary surfaces:

- existing OWL class declarations;
- existing OWL object property declarations;
- existing EKCD-local datatype property declarations;
- existing OWL annotation property declarations;
- existing EKCD metadata and source-reference conventions, except for normal version metadata updates.

Therefore, v1.1.2 should be treated as a **non-breaking vocabulary-expansion release**. Existing ABox data using v1.0.8 terms should remain valid, but new ABox data may use the added interpretive-assertion, confidence-level, evidence-tracking, and utterance-modeling terms.

## 9. Impact on ABox data

Existing ABox data should not require term replacement. However, maintainers may revise or extend ABox modeling practices as follows:

| Area | v1.1.2 recommendation |
|---|---|
| Resource IRIs | Continue using explicit `ekc:`, `ekcd:`, and project-specific prefixed names. |
| Empty prefix | Continue avoiding the empty prefix `:`. |
| Relative IRIs | Continue avoiding relative forms such as `<#...>`, `<...>`, or `<>`. |
| URL literals | Keep concrete URLs on target web-resource nodes using `ekcd:accessURL`. |
| Display links | Keep `edm:isShownAt` and `edm:isShownBy` as object properties. |
| Date expressions | Use `ekcd:occurrenceDate`, `ekcd:startDate`, `ekcd:endDate`, and `ekcd:originalDate` according to single-date/period/original-source needs. |
| Interpretive relations | Use `ekcd:InterpretiveAssertion` when a correspondence or contextual relation needs explicit subject, predicate, object, evidence, and confidence modeling. |
| Correspondence | Use `ekcd:correspondsTo` for scene-procedure, record-event, or source-unit correspondences; do not use it as a substitute for `owl:sameAs`. |
| Contextual association | Use `ekcd:contextuallyRelatedTo` for broader historical, textual, visual, or interpretive association. |
| Confidence levels | Use `ekcd:Direct`, `ekcd:Probable`, and `ekcd:Contextual` as controlled values through `ekcd:hasInterpretiveConfidence`. |
| Evidence tracking | Use `prov:wasDerivedFrom`, `ekcd:evidenceLocation`, and `ekcd:evidenceNote` to connect interpretive assertions to their source materials and evidence explanations. |
| Utterance modeling | Model utterances as `ekcd:Utterance` or its subclasses, and connect them to speakers, records, topics, and retrospectively framed events using the new utterance properties. |
| Record separation | Continue modeling the record that documents an utterance separately as an instance of `ekc:Record` or its subclass; do not collapse the utterance event and the documentary record into one node. |
