## 5. Ontology metadata changes

### 5.1 Added vocabulary metadata

The ontology resource `http://dh.aks.ac.kr/ontologies/ekcd` is now also typed as a VOAF vocabulary:

```turtle
<http://dh.aks.ac.kr/ontologies/ekcd> rdf:type owl:Ontology ,
                                               voaf:Vocabulary .
```

The release also adds VANN namespace metadata:

```turtle
vann:preferredNamespacePrefix "ekcd" ;
vann:preferredNamespaceUri "http://dh.aks.ac.kr/ontologies/ekcd#" .
```

This strengthens the repository-level description of EKCD as a reusable vocabulary and makes the preferred namespace explicit for publication, discovery, and citation contexts.

### 5.2 Prefix declaration changes

`v1.0.7` adds:

```turtle
@prefix vann: <http://purl.org/vocab/vann/> .
```

The unused bare prefix declaration below was removed:

```turtle
@prefix : <http://dh.aks.ac.kr/ontologies/ekc#> .
```

This change improves serialization clarity without changing any EKCD term IRI.

### 5.3 Title and description updates

The ontology title was revised from a shorter personal-extension framing to a clearer public-release title:

| Language | v1.0.4 | v1.0.7 |
|---|---|---|
| English | `EKCD Ontology: EKC-derived personal extension ontology` | `EKCD: An EKC-derived Ontology for Korean Cultural Heritage Knowledge Modeling` |
| Korean | `EKCD 온톨로지: EKC 파생 개인 확장 온톨로지` | `EKCD: 한국 문화유산 지식 모델링을 위한 EKC 파생 온톨로지` |

The description was also revised to emphasize:

- EKC-derived continuity;
- Korean cultural heritage knowledge modeling;
- date and URL datatype-property support;
- local application guidelines for standard semantic mapping properties;
- project-level semantic extensions.

## 6. Datatype-property description revision

### 6.1 Changed term

`ekcd:occurrenceDate`

### 6.2 Nature of the change

The Korean description was revised to use explicit `ekcd:` prefix references for related EKCD datatype properties.

#### v1.0.4 wording

```text
기간 사건의 경우에는 :startDate와 :endDate를 우선 사용하고, 원사료형 날짜 표기는 :originalDate에 가능한 한에서 기록한다.
```

#### v1.0.7 wording

```text
기간 사건의 경우에는 ekcd:startDate와 ekcd:endDate를 우선 사용하고, 원사료형 날짜 표기는 ekcd:originalDate에 가능한 한에서 기록한다.
```

### 6.3 Modeling impact

This is a documentation-level correction. It clarifies that the related properties belong to the EKCD namespace and avoids confusion with the removed bare `:` prefix. The change does not affect the formal IRI, datatype range, label, or intended modeling function of `ekcd:occurrenceDate`.

## 7. Backward compatibility assessment

`v1.0.7` is backward-compatible with `v1.0.4`.

- Existing EKCD term IRIs remain stable.
- No classes or properties were renamed.
- No classes or properties were removed.
- No object-property semantics were changed.
- No datatype-property ranges were changed.
- The changes improve publication metadata and descriptive clarity only.
